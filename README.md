# ИИ: синхронизация — CEP-плагин для Premiere Pro

Автоматическая синхронизация клипов многокамерного материала по общему звуку
(кросс-корреляция RMS-огибающих) с по-клипной точностью ±1 кадр, ripple-закрытием
пауз, поддержкой разных fps источников и линейной коррекцией дрейфа часов записи.

Плагин самостоятельный, не часть «ИИ: монтаж».

## Возможности

- **По-клипная синхронизация** против опорной (anchor) дорожки — работает при
  перезапусках камер (много клипов на дорожку).
- **Превью перед применением**: таблица (сдвиг, статус, confidence) + canvas-вейвформ
  огибающих (клик по клипу).
- **Confidence-гейт**: ненадёжные матчи помечаются, не двигаются вслепую.
- **Ripple-закрытие пауз** после сдвигов.
- **Разные fps** источников (вся математика в домене времени → тики).
- **Коррекция дрейфа** (5–50 ppm на длинных подкастах): детекция двумя корреляциями,
  rate-stretch по порогу.
- **Безопасность**: checkpoint-секвенция перед Apply, Revert одним кликом.

## Требования

- Adobe Premiere Pro 2024+ (Host PPRO `[24.0,99.9]`, CSXS 12).
- **ffmpeg** в PATH или `C:\ffmpeg\bin\ffmpeg.exe`.
- Node.js ≥ 18 (для запуска тестов).
- Включённый CEP **PlayerDebugMode** (для неподписанного расширения).

## Установка (dev)

Симлинк/junction репозитория в папку расширений CEP:

```powershell
New-Item -ItemType Junction `
  -Path "$env:APPDATA\Adobe\CEP\extensions\com.gleb.aisync" `
  -Target "C:\Users\Глеб\Documents\Sync_Premier"
```

Перезапустить Premiere (чтобы он обнаружил расширение), затем открыть
**Window → Extensions → «ИИ: синхронизация»**.

## Отладка

Debug-порт **8100** (CDP):

```bash
CEP_DEBUG_PORT=8100 node tools/cep-debug.mjs targets
CEP_DEBUG_PORT=8100 node tools/cep-debug.mjs host '$._SYNC_.getTimelineSnapshot()'
CEP_DEBUG_PORT=8100 node tools/cep-debug.mjs reload
```

## Тесты

```bash
npm test
```

Юнит-тесты ядра (vm-loader): кросс-корреляция, confidence-гейт, детекция дрейфа,
RMS-огибающая, ремап дорожек, выбор опоры, сборка огибающей опоры, downsample вейвформа.

## Цикл работы

1. **Анализировать таймлайн** — снимок секвенции, выбор опоры, по-клипный матчинг.
2. Проверить превью (таблица + вейвформ по клику на клип).
3. **Применить** — checkpoint → сдвиги → ripple → коррекция дрейфа.
4. **Откатить** — вернуться на checkpoint-секвенцию.
